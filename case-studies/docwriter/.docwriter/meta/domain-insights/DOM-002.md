## Insight

Xperience documentation uses manual hub/index pages (e.g., content-item-api.md) that enumerate sub-pages in prose. When new pages are created in a directory, the parent index page must be updated manually. The cross-ref-updater only verifies existing page_link references — it does not detect missing enumeration entries on hub pages. Future runs creating new pages should auto-generate an impact for the parent directory's index page.

## Evidence

### Run 2 (DOC-3137)

- Run 2 (DOC-3137): T-004 (hub page update) and T-005 (set-up-activities update) both addressed hub enumeration correctly. DOM-002 guidance applied successfully.
- Acceptance rate across 2 runs: confirmed
- Promoted: low → medium (2nd pipeline run confirmation)

### Run 1 (original)

- Source signal: CTXGAP-002 (gap analysis) + CTXDOM-002 (domain insight analysis) — converging signals from two independent analyzers
- Within-run strength: moderate (2 independent observations from different analysis passes)
- GAP-002 found content-item-api.md (hub page) did not list the new reference-content-access-settings.md
- Cross-ref-updater added 3 new page_links but did not detect the missing hub enumeration
- The verification matrix shows 477 outbound links verified but this is link-validation, not completeness-checking of hub page inventories
- Confidence: low (cold-start run — converging within-run evidence, but needs second run confirmation for medium)

## Recommendation

When the impact mapper identifies a create-action (new page) task, automatically generate a secondary impact for the parent directory's hub/index page. The cross-ref-updater should be extended to check hub page enumeration completeness, not just link validity.

## Applicability

Applicable to any Xperience documentation pipeline run that creates new pages. Hub pages exist for API reference areas, how-to guide collections, and conceptual overview sections. This pattern will recur whenever the change inventory includes new API surfaces requiring new documentation pages.
