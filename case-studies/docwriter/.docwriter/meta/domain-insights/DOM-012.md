## Insight

Xperience has high-connectivity documentation hub pages (e.g., reference-global-system-events with 17+ outgoing links) that connect to multiple product areas. When new extensibility point documentation is created, bidirectional cross-referencing to/from these hubs is mandatory. The cross-ref updater's outgoing-link check catches forward links but the content writer may miss the return link.

## Evidence

- GAP-002: T-010 (reference-global-system-events) linked to T-001 but T-001 did not link back
- reference-global-system-events connects to activity logging, contact management, email marketing, and form processing
- Cross-ref updater verified outgoing links but relied on content writer for initial bidirectional set
- Signal strength: medium
- Confidence: low (first observation in a single run)

## Recommendation

Impact mapper should flag high-connectivity pages (>15 links) with mandatory bidirectional cross-ref requirements when new pages reference them. The content writer's acceptance criteria should include "verify bidirectional cross-ref with [hub page]".

## Applicability

Applicable to any pipeline run creating new extensibility or feature documentation that intersects with existing hub pages.
