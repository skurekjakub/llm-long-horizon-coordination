## Insight

Style guide source files can be safely deleted when the invariant inventory extracted from them (plus other guideline sources) fully covers their enforcement surface. The invariant scanner tracks source file hashes and correctly handles deletions by mapping them to 0 invariants. No orphaned references or enforcement gaps result.

In DOC-2847, intentional deletion of `ai-agent-invariants.md` and replacement with `docwriter-agent-invariants.md` (9 new invariants) was processed cleanly. The invariant scanner's `invariant-hashmap.json` tracked 14 source files — all present on disk after the swap. Total invariant surface: 146 enforceable rules across 15 source files. The gap hunter confirmed no orphaned references.

The prompt template file `.github/prompts/EXAMPLE-update-code-samples-in-guides.md` was also deleted with zero impact — it was not a guidelines source file.

## Applicability

Applicable when consolidating or pruning guideline sources. Before deleting a style guide file, verify that its enforceable rules have been extracted into the invariant inventory. After deletion, run invariant scanner to confirm hashmap convergence and gap hunter to verify no orphaned references. This is a meta-process insight for pipeline maintenance, not content documentation.
