## Insight

When a docFact (code-analysis finding) mandates using an API that resides in an .Internal namespace, it conflicts with INV-general-012 (prohibiting documentation of .Internal APIs). The resolution is invariant supremacy: discard the docFact, substitute a public alternative, and document the reasoning. In DOC-3167, IDateTimeNowService (CMS.Core.Internal) was mandated by FIND-051 but the writer correctly substituted DateTime.UtcNow with a testability tip recommending .NET TimeProvider.

This conflict pattern (internal-namespace testability API) may recur for other .Internal utilities in the Xperience platform.

## Evidence

- DOC-3167 T-001: IDateTimeNowService mandated by FIND-051 but lives in CMS.Core.Internal namespace. Writer applied invariant supremacy, discarded the docFact, substituted DateTime.UtcNow. Resolution accepted on attempt 2.
- Accuracy reviewer confirmed the resolution was correct.
- The conflict was the root cause of one of T-001's two rejection categories (accuracy).

## Applicability

Applicable to any documentation task where code analysis findings reference types in .Internal namespaces. The resolution pattern (discard docFact, substitute public API, add testability tip) generalizes to any invariant-vs-finding conflict where the invariant has higher authority.
