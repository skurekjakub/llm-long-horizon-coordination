## Insight

Xperience admin roles (RoleInfo/cms.role) and membership roles (MemberRoleInfo/cms.memberrole) are entirely separate systems with different database tables, Info classes, and management APIs. Documentation touching 'roles' must always disambiguate which role system is being discussed. In DOC-3167, T-001 included an explicit info box distinguishing admin vs member roles — this disambiguation was essential for all 11 role-referencing tasks.

## Evidence

- DOC-3167: FIND-004 documents RoleInfo for admin users. FIND-050 is a cross-cutting constraint requiring explicit distinction.
- T-001 included an explicit info box distinguishing admin vs member roles.
- 11 of 18 tasks (61%) in this run referenced role concepts and needed consistent disambiguation.

## Applicability

Applicable to any documentation task involving Xperience roles, whether admin or membership. The disambiguation is especially important in overview and getting-started pages where readers may not yet understand the platform's dual-role architecture. Related to but distinct from DOM-004 (content security tiers).
