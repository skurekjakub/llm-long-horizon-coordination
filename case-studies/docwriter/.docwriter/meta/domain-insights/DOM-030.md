## Insight

Xperience scheduled task DI works via ActivatorUtilities.CreateInstance — any registered service can be constructor-injected, making scheduled tasks full DI citizens rather than isolated utilities. Documentation for scheduled tasks should show constructor injection of platform services (IInfoProvider, IEventLogService, etc.) as the standard pattern, not a special case.

## Evidence

- DOC-3167: FIND-016 documents SchedulingExecutor using ActivatorUtilities.CreateInstance(serviceProvider, scheduledTask.ScheduledTaskType).
- T-001 demonstrated this by injecting IInfoProvider<MemberRoleMemberInfo>, IEventLogService, and other services.
- Reference implementations (FIND-017 RecycleBinCleanerTask, FIND-018 ContentItemScheduledPublishAndUnpublishTask) confirm the pattern.
- FIND-051 synthesizes the DI pattern across 3 reference tasks.

## Applicability

Applicable to any documentation task involving Xperience scheduled tasks. When documenting custom scheduled tasks, show constructor injection as the primary DI mechanism and reference platform reference implementations as examples.
