## Insight

Tasks applying the full INV-codesamples-* invariant suite (10 invariants including INV-codesamples-004 build verification and INV-codesamples-011 named parameter rule) combined with PAT-001 and REC-003/010/011/012 achieve 100% first-attempt acceptance rate for mechanical code-sample transformations. The mechanical nature of the conversion plus compile-time verification eliminates the accuracy errors seen in T-001/T-002 where pre-existing code was trusted without verification.

Key invariant anchors:
- `INV-codesamples-004`: Build verification — catches non-existent method calls at compile time
- `INV-codesamples-011`: Named parameter rule — 4+ parameter calls use named syntax

## Applicability

Applicable to any task requiring systematic code-sample transformation (positional→named, syntax migration, API call updates) where the target files are compilable (.cs, .ts, etc.) and the transformation is mechanical (not semantic). Less applicable to .md documentation code blocks where build verification cannot run. The full INV-codesamples-* suite provides the safety net; partial application is untested.
