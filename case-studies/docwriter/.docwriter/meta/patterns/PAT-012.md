## Insight

When inserting a new callout or note into an existing documentation page, mirroring the exact structure and phrasing of an existing callout on the same page achieves first-attempt acceptance. The existing content serves as a pre-validated template — its style, formatting, and structural conventions have already passed review. This reduces reviewer friction because the new content is structurally consistent with its surroundings.

## Evidence

- DOC-3167 T-004: Inserted permission note for 'Clone a page' section by matching the exact structural pattern of the existing 'Create a new page' permission note on the same page. First-attempt acceptance.
- DOC-3167 T-008: Inserted permission cross-reference note by matching existing note patterns. First-attempt acceptance.
- Both tasks had narrow scope (5 acceptance criteria each) and explicitly mirrored existing content structure.
- Acceptance rate: 100% (2/2 tasks using this approach)

### Run 4 (DOC-3167 continued)

- T-007: Used PAT-012 to mirror existing page structure when inserting info callout in retrieve-page-content.md. Reviewer approved on first attempt (multi-cycle was for voluntary enhancement, not rejection).
- T-017: Used PAT-012 to mirror existing page structure for tip callout in handle-global-events.md. First-attempt acceptance.
- Both tasks explicitly cited PAT-012. Combined with Run 3 evidence (2 tasks, 100%), now 4 tasks with 100% first-attempt reviewer approval.
- **Promoted: low → medium** (2nd pipeline run confirmation, 100% first-attempt reviewer approval across 4 tasks)

## Applicability

Applicable to any update task where new structured content (callouts, notes, info boxes, warning blocks) is inserted into a page that already contains similar elements. Confirmed across 2 pipeline runs. Less applicable when creating entirely new pages or when the existing content on the page has known style issues.
