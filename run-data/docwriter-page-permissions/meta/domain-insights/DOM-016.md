## Insight

In gap convergence cycles, structural fixes (page rewrites, convention changes) have higher cascade risk than content fixes (text corrections, cross-ref additions). The DOC-3137 convergence pattern [2, 1, 1] masked a qualitative escalation: cycle 1 gaps were content-level (directive framing, missing cross-ref), cycle 2 was structural (_api convention rewrite), cycle 3 was a cascade from cycle 2's structural fix (link text invalidated by rewrite). Content fixes resolve cleanly; structural fixes can create new gaps.

## Evidence

- GAP-001 (content fix) → resolved cleanly in cycle 2, no cascade
- GAP-002 (content fix) → resolved cleanly in cycle 2, no cascade
- GAP-003 (structural fix) → resolved in cycle 3 but cascaded into GAP-004
- GAP-004 (cascade from structural fix) → force-converged at cycle 3 safety valve
- Signal strength: medium
- Confidence: low (first observation in a single run)

## Recommendation

When a gap fix requires a structural rewrite, add a cascade-risk assessment before re-entering the pipeline. Structural fixes should trigger re-validation of all incoming cross-references, link texts, and content promises on referring pages.

## Applicability

Applicable to any pipeline run where gap hunting discovers structural convention violations that require page rewrites.
