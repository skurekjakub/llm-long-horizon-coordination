## Insight

Narrow-scope tasks that modify only 1-2 code blocks and require only 2-4 invariants consistently pass first-attempt review. No meta-knowledge patterns or research recommendations are needed — the scope is small enough that invariant coverage is naturally sufficient. These tasks represent the lowest-complexity tier and are essentially risk-free for the pipeline.

## Evidence

- DOC-2847 T-006: Single code block named parameter update, 4 invariants applied, first-attempt acceptance
- DOC-2847 T-010: Two code blocks on same page, 2 invariants applied, first-attempt acceptance (consistency enforcement with already-converted sibling blocks)
- Acceptance rate: 100% (2/2 tasks)

## Applicability

Applicable to any task with ≤2 code block modifications and no prose/section restructuring. The pattern degrades when prose updates, cross-references, or callout formatting are added (those trigger style micro-invariant exposure as seen in T-008). Use this pattern to identify "safe" tasks in the task graph that can be deprioritized for review cycle budgeting.
